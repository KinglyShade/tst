Items ConfigurationConfiguración de Objetos

This section explains how to configure individual items in items.yml, which can be used as boss rewards, equipment, or other purposes, with custom properties and enchantments.

Esta sección explica cómo configurar objetos individuales en items.yml, que pueden usarse como recompensas de jefes, equipamiento u otros fines, con propiedades y encantamientos personalizados.

1. Basic Structure1. Estructura Básica

Items are defined under the 'items' key with a unique identifier, often a timestamp.

Los objetos se definen bajo la clave 'items' con un identificador único, a menudo un timestamp.

items:
item_1650972105000:
# Item settings go here

Explanation: The items key is the root for all item definitions. Each item (e.g., item_1650972105000) uses a unique identifier, typically a timestamp, to avoid conflicts and track items easily.

Explicación: La clave items es la raíz de todas las definiciones de objetos. Cada objeto (por ejemplo, item_1650972105000) usa un identificador único, típicamente un timestamp, para evitar conflictos y rastrear ítems fácilmente.

2. Core Item Settings2. Configuraciones Básicas de Objetos

Define the item’s type, name, enchantments, and lore.

Define el tipo, nombre, encantamientos y lore del objeto.

item_1650972105000:
type: "DIAMOND_SWORD"
name: "Espada de Diamante"
enchanted_effect: false
enchanted: true
lore:
- "Un ítem muy poderoso."
- "Desata tu verdadero potencial."
enchants:
DAMAGE_ALL: 5
DURABILITY: 3

Explanation:

  • type: "DIAMOND_SWORD": Minecraft item type (e.g., DIAMOND_SWORD, BOW, SHIELD).
  • name: "Espada de Diamante": Custom display name, supports color codes.
  • enchanted_effect: false: If true, shows enchantment glow even without enchants; false hides it unless enchanted.
  • enchanted: true: Indicates the item has enchantments.
  • lore: List of descriptive lines shown in the item tooltip.
  • enchants: Enchantments with levels (e.g., DAMAGE_ALL: 5 is Sharpness V, DURABILITY: 3 is Unbreaking III).

Explicación:

  • type: "DIAMOND_SWORD": Tipo de ítem de Minecraft (por ejemplo, DIAMOND_SWORD, BOW, SHIELD).
  • name: "Espada de Diamante": Nombre personalizado, soporta códigos de color.
  • enchanted_effect: false: Si es true, muestra el brillo de encantamiento incluso sin encantamientos; false lo oculta a menos que esté encantado.
  • enchanted: true: Indica que el ítem tiene encantamientos.
  • lore: Lista de líneas descriptivas mostradas en la descripción del ítem.
  • enchants: Encantamientos con niveles (por ejemplo, DAMAGE_ALL: 5 es Sharpness V, DURABILITY: 3 es Unbreaking III).

Main ConfigurationConfiguración Principal

This section explains the main settings in config.yml, which controls the plugin’s general behavior, boss spawning, health bars, abilities, and permissions.

Esta sección explica las configuraciones principales en config.yml, que controla el comportamiento general del plugin, la aparición de jefes, barras de salud, habilidades y permisos.

1. General Settings1. Configuraciones Generales

Basic plugin settings for language, updates, and debugging.

Configuraciones básicas del plugin para idioma, actualizaciones y depuración.

settings:
  language: "en_us"
  check-for-updates: true
  debug-mode: false

Explanation:

  • check-for-updates: true: Notifies admins of new plugin versions.
  • debug-mode: false: When true, logs detailed debug info for troubleshooting.

Explicación:

  • language: "en_us": Establece el idioma para los mensajes del plugin (por ejemplo, "en_us" para inglés, "es_es" para español).
  • check-for-updates: true: Notifica a los administradores sobre nuevas versiones del plugin.
  • debug-mode: false: Cuando es true, registra información detallada de depuración.

Controls logging for debugging purposes.

Controla el registro para fines de depuración.

logging:
  enable-logs: true
  log-file: "logs/kinglybosses.log"

Explanation:

  • enable-logs: true: Enables logging of plugin events.
  • log-file: "logs/kinglybosses.log": Path to the log file relative to the server directory.
Note: Logging is in beta and may be refined in future updates.

Explicación:

  • enable-logs: true: Habilita el registro de eventos del plugin.
  • log-file: "logs/kinglybosses.log": Ruta al archivo de registro relativa al directorio del servidor.
Nota: El registro está en beta y puede refinarse en futuras actualizaciones.

Controls how bosses spawn globally.

Controla cómo aparecen los jefes globalmente.

general:
  spawnRandom: true
  spawnInterval: 300
  broadcastBossSpawn: true
  location:
    world: "world"
    coordinates:
      x: 0
      y: 0
      z: 0

Explanation:

  • spawnRandom: true: Enables random spawning (currently the only mode, temporarily disabled for other options).
  • spawnInterval: 300: Time between random spawns (300 seconds = 5 minutes).
  • broadcastBossSpawn: true: Announces boss spawns to all players.
  • location: Default spawn point if no specific location is set in boss.yml; uses "world" at coordinates (0, 0, 0).

Explicación:

  • spawnRandom: true: Habilita la aparición aleatoria (actualmente el único modo, deshabilitado temporalmente para otras opciones).
  • spawnInterval: 300: Tiempo entre apariciones aleatorias (300 segundos = 5 minutos).
  • broadcastBossSpawn: true: Anuncia las apariciones de jefes a todos los jugadores.
  • location: Punto de aparición predeterminado si no se establece una ubicación específica en boss.yml; usa "world" en las coordenadas (0, 0, 0).

Customizes the boss health bar display.

Personaliza la visualización de la barra de salud del jefe.

bossHealthBar:
  active: true

  color: "RED"
  style: "SOLID"

  update_interval: 0.1
  display_distance:
    enabled: true
    radius: 100
  permission_based_display:
    enabled: false
    permission: "bossbar.view"
  hide_on_death:
    enabled: true

Explanation:

  • active: true: Enables the health bar.
  • title: Displayed text; {boss} is replaced by the boss’s name.
  • color: "RED": Bar color (options: red, blue, green, etc.).
  • style: "SOLID": Bar style (e.g., solid, segmented_6).
  • progress: 1.0: Initial health percentage (0 to 1).
  • update_interval: 0.1: Updates every 0.1 seconds.
  • display_distance: Limits visibility to players within 100 blocks.
  • permission_based_display: When enabled, restricts visibility to players with the "bossbar.view" permission.
  • hide_on_death: true: Hides the bar when the boss dies.

Explicación:

  • active: true: Habilita la barra de salud.
  • title: Texto mostrado; {boss} se reemplaza por el nombre del jefe.
  • color: "RED": Color de la barra (opciones: red, blue, green, etc.).
  • style: "SOLID": Estilo de la barra (por ejemplo, solid, segmented_6).
  • progress: 1.0: Porcentaje inicial de salud (0 a 1).
  • update_interval: 0.1: Se actualiza cada 0.1 segundos.
  • display_distance: Limita la visibilidad a jugadores dentro de 100 bloques.
  • permission_based_display: Cuando está habilitado, restringe la visibilidad a jugadores con el permiso "bossbar.view".
  • hide_on_death: true: Oculta la barra cuando el jefe muere.

Define las habilidades globales de los jefes.

  • enableBossAbilities: true: Activates abilities (temporarily disabled).
  • abilityCooldown: 10: Default cooldown between ability uses (10 seconds).
  • abilityList: Predefined abilities available to bosses; specific behaviors depend on implementation.

Explicación:

  • enableBossAbilities: true: Activa las habilidades (deshabilitado temporalmente).
  • abilityCooldown: 10: Enfriamiento predeterminado entre usos de habilidades (10 segundos).
  • abilityList: Habilidades predefinidas disponibles para los jefes; los comportamientos específicos dependen de la implementación.

Adds dynamic combat features like enrage and phases.

Añade características dinámicas de combate como furia y fases.

advancedMechanics:
  enrage:
    enabled: true
    trigger_health_percentage: 20
    effect:
      particle: "FLAME"
      sound: "ENTITY_ENDER_DRAGON_GROWL"
    damage_multiplier: 1.5
  phases:
    enabled: true
    phaseList:
      - health_percentage: 50
        message: "&cThe boss grows stronger!"
        abilities: ["lightning", "summon_minions"]
      - health_percentage: 25
        message: "&cThe boss is in a frenzy!"
        abilities: ["fireball", "teleport"]

Explanation:

  • enrage: Boosts the boss at low health.
  • trigger_health_percentage: 20: Enrages at 20% health.
  • effect: Adds visual (flame particles) and audio (dragon growl) cues.
  • damage_multiplier: 1.5: Increases damage by 50% when enraged.
  • phases: Changes boss behavior at health thresholds.
  • phaseList: Defines phases; at 50% health, uses lightning and minions; at 25%, uses fireball and teleport.

Explicación:

  • enrage: Potencia al jefe con poca salud.
  • trigger_health_percentage: 20: Se enfurece al 20% de salud.
  • effect: Añade señales visuales (partículas de fuego) y auditivas (rugido de dragón).
  • damage_multiplier: 1.5: Aumenta el daño en un 50% cuando está enfurecido.
  • phases: Cambia el comportamiento del jefe en umbrales de salud.
  • phaseList: Define fases; al 50% de salud, usa relámpagos y minions; al 25%, usa bola de fuego y teletransporte.

Item Groups ConfigurationConfiguración de Grupos de Objetos

This section explains how to configure item groups in itemgroups.yml, which define sets of items that can be awarded as boss rewards with specified probabilities, referencing items from items.yml.

Esta sección explica cómo configurar grupos de objetos en itemgroups.yml, que definen conjuntos de ítems que pueden otorgarse como recompensas de jefes con probabilidades especificadas, haciendo referencia a ítems de items.yml.

1. Basic Structure1. Estructura Básica

Item groups are defined under the 'itemgroups' key with a unique identifier.

Los grupos de objetos se definen bajo la clave 'itemgroups' con un identificador único.

itemgroups:
grupo1:
# Item group settings go here

Explanation: The itemgroups key is the root for all item group configurations. Each group (e.g., grupo1) is a unique identifier that organizes a set of items and their drop mechanics.

Explicación: La clave itemgroups es la raíz de todas las configuraciones de grupos de objetos. Cada grupo (por ejemplo, grupo1) es un identificador único que organiza un conjunto de ítems y sus mecánicas de caída.

2. Core Item Group Settings2. Configuraciones Básicas de Grupos de Objetos

Define the number of rolls and the items with their probabilities.

Define el número de tiradas y los ítems con sus probabilidades.

grupo1:
rolls: 1
items:
- item: "item_1650972105000"
probability: 70
- item: "item_1650972105000"
probability: 30

Explanation:

  • rolls: 1: Number of times the game randomly selects an item from this group per reward event.
  • items: List of possible items in the group, referencing IDs from items.yml.
  • item: "item_1650972105000": Unique identifier for an item defined in items.yml.
  • probability: 70: Chance (in percent) of this item being selected per roll; here, the same item has two entries with different probabilities.

Explicación:

  • rolls: 1: Número de veces que el juego selecciona aleatoriamente un ítem de este grupo por evento de recompensa.
  • items: Lista de ítems posibles en el grupo, haciendo referencia a IDs de items.yml.
  • item: "item_1650972105000": Identificador único de un ítem definido en items.yml.
  • probability: 70: Probabilidad (en porcentaje) de que este ítem sea seleccionado por tirada; aquí, el mismo ítem tiene dos entradas con diferentes probabilidades.

Ejemplos completos de diferentes grupos de objetos con tiradas y probabilidades variadas.

Explanation:

  • grupo2: Allows 2 rolls with balanced probabilities (total 100%), offering a mix of items like the bow, axe, and shield from items.yml.
  • rare_group: Single roll with a rare sword (10%) and a more common shield (90%), emphasizing rarity.
  • legendary_group: 3 rolls with high-value items (e.g., blade, armor, ring), probabilities summing to 100%, increasing reward potential.
  • Probabilities don’t need to sum to 100% per group; the system may normalize them or allow over/under based on plugin mechanics.

Explicación:

  • grupo2: Permite 2 tiradas con probabilidades balanceadas (total 100%), ofreciendo una mezcla de ítems como el arco, hacha y escudo de items.yml.
  • rare_group: Una sola tirada con una espada rara (10%) y un escudo más común (90%), enfatizando la rareza.
  • legendary_group: 3 tiradas con ítems de alto valor (por ejemplo, espada, armadura, anillo), probabilidades que suman 100%, aumentando el potencial de recompensa.
  • Las probabilidades no necesitan sumar 100% por grupo; el sistema puede normalizarlas o permitir más/menos según las mecánicas del plugin.

⚠️ Tips & Common Errors⚠️ Consejos y Errores Comunes

  • Ensure item IDs match those defined in items.yml or valid Minecraft item names.
  • Asegúrate de que los IDs de item coincidan con los definidos en items.yml o nombres de ítems válidos de Minecraft.
  • Adjust rolls and probability to balance reward frequency and rarity.
  • Ajusta rolls y probability para equilibrar la frecuencia y la rareza de las recompensas.
  • Test groups with high rolls to avoid overwhelming players with items.
  • Prueba grupos con rolls altos para evitar abrumar a los jugadores con ítems.

Groups are defined under the 'groups' key with a unique identifier.

Los grupos se definen bajo la clave 'groups' con un identificador único.

groups:
